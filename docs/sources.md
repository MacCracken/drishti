# drishti — Sources

> Last Updated: 2026-07-10

The citation index for drishti's algorithmic and domain content. AGNOS
codec crates carry a sources file so every nontrivial parse/coding step
is traceable to a spec or primary reference, and so a future maintainer
can re-derive (or audit) any of it. Entries are grouped by module
family. Discipline: the SPEC is the derivation; implementations are
cross-checks — multiple of them, never a single source
(redesign-don't-reinvent).

## Core (`src/drishti.cyr`, `src/bits.cyr`, `src/ivf.cyr`)

- **AV1 Bitstream & Decoding Process Specification** v1.0.0 w/ Errata 1
  (AOMedia) — 4.10.3 uvlc(), 4.10.5 leb128() (the ≤8-byte / ≤32-bit
  conformance bounds `dr_leb128_read` enforces).
- **ITU-T Rec. H.264** — 9.1/9.1.1 ue(v)/se(v) exp-Golomb parsing
  process (shared verbatim by H.265 9.2) — `dr_ue_read`/`dr_se_read`
  and the writer inverses.
- **IVF container** — the de-facto libvpx/AOM test-bench format
  (32-byte `DKIF` header + 12-byte frame headers); layout cross-checked
  against libvpx `ivfdec.c`/`ivfenc.c` and dav1d's `tools/input/ivf.c`.

- **AV1 reconstruct process** — spec section 7.12.3 is authoritative (`src/av1_recon.cyr`): the per-coefficient dequantization (`dq = Quant·q`, `dq2 = sign·((|dq| & 0xFFFFFF)/dqDenom)`, the `±2^(7+BitDepth)` clamp), `dqDenom` (2 for the 32-class, 4 for the 64-class transforms), the 2D inverse transform invocation (7.13.3), and the `flipUD`/`flipLR` (FLIPADST) residual add with `Clip1`. An adversarial review verified the dequant math, the flip/dqDenom, the transform integration, and the known-answers (reusing the inverse-transform DC/IDTX/WHT vectors as oracles).

- **AV1 default coefficient CDFs** — the additional-tables annex is authoritative (`src/av1_coeffcdf.cyr`): all seven initial adaptive-CDF families — `Default_Txb_Skip_Cdf`, `Default_Eob_Pt_{16..1024}_Cdf`, `Default_Eob_Extra_Cdf`, `Default_Dc_Sign_Cdf`, `Default_Coeff_Base_Eob_Cdf`, `Default_Coeff_Base_Cdf`, `Default_Coeff_Br_Cdf` — across the 4 `COEFF_CDF_Q_CTXS` quantizer buckets (15,996 entries). Stored in the symbol coder's forward-cumulative format (N freqs ending in 32768 + a 0 count). Extracted from the spec (evaluating the `128 * x` products) and cross-checked against **libaom** `av1/common/token_cdfs.h` (accounting for libaom's AOM_CDF inverse-value convention); adversarial reviews diffed every value and validated CDF structure.
- **AV1 coefficient reading loop** — spec section 5.11.39 (`coeffs()`) is authoritative (`src/av1_coeffs.cyr`), with the CDF-selection contexts from 8.3.2 (`all_zero`/txb_skip and `dc_sign`) and the `Tx_Size_Sqr` / `Tx_Size_Sqr_Up` tables + `txSzCtx`. `av1_coeffs_decode` reads all_zero, the eob position (`eob_pt` + `eob_extra`), the base levels (`coeff_base` / `coeff_base_eob`), the `coeff_br` range extension, the signs (`dc_sign` / `sign_bit`), and the Exp-Golomb tail; `av1_coeffs_encode` is its exact inverse. Cross-checked against **libaom** `av1/decoder/decodetxb.c` (`av1_read_coeffs_txb`, `read_golomb`) and the reference eob/base/br/golomb control flow. The unary golomb length is capped (libaom-style) so a truncated stream can never hang. Verified by decode/encode round-trip. The **adaptive per-tile CDF context** (`av1_ccdf_*`, mirroring libaom `init_coeff_cdfs` + the per-tile CDF copy that the symbol coder adapts in place) lets `coeffs()` run with `disable_cdf_update = 0`; verified by an adaptive multi-block round-trip.
- **AV1 default non-coefficient CDFs** — the additional-tables annex is authoritative (`src/av1_noncoeffcdf.cyr`): the intra-keyframe default CDF tables — `Default_Partition_W{8,16,32,64}_Cdf`, `Default_Skip_Cdf`, `Default_Intra_Frame_Y_Mode_Cdf`, `Default_Uv_Mode_Cfl_{Not_}Allowed_Cdf`, `Default_Angle_Delta_Cdf`, `Default_Cfl_Sign_Cdf`, `Default_Cfl_Alpha_Cdf`, `Default_Filter_Intra{_Mode}_Cdf`, `Default_Tx_{8x8,16x16,32x32,64x64}_Cdf`, `Default_Intra_Tx_Type_Set{1,2}_Cdf` (1,622 entries). Stored in the symbol coder's forward-cumulative format; extracted from the spec (evaluating `128 * x`) and cross-checked against **libaom** `av1/common/entropymode.c` (accounting for the AOM_CDF inverse-value convention). The block/partition-decode reads that consume these land in later 0.7.x bites. The 7-bite block-decode decomposition was mapped by a multi-agent scoping pass.
- **AV1 intra mode-info reads** — spec section 5.11.16 (`intra_frame_mode_info`) is authoritative (`src/av1_modeinfo.cyr`), with the CDF-selection contexts from the parsing process (spec 9.3): `read_skip` (ctx = `Skips[above]+Skips[left]`), `intra_frame_y_mode` (cdf `[abovemode][leftmode]`, `abovemode`/`leftmode` = `Intra_Mode_Context[avail ? neighbourYMode : DC_PRED]`), `intra_angle_info_y`/`_uv` (angle_delta gated on `MiSize>=BLOCK_8X8` + `is_directional_mode`, cdf index `mode-V_PRED`), `uv_mode` (the Lossless / `Max(BW,BH)<=32` CfL-allowed cdf selection via `get_plane_residual_size`), `read_cfl_alphas` (joint sign `signU=(signs+1)/3`, `signV=(signs+1)%3`; magnitude ctx `(signU-1)*3+signV` / `(signV-1)*3+signU`, spec 5.11.45), and `filter_intra_mode_info` (`enable_filter_intra && YMode==DC_PRED && Max(BW,BH)<=32`). Decode functions plus their exact-inverse encoders (round-trip tested through the symbol coder). Ships with the block-size conversion tables — `Mi_Width/Height_Log2`, `Num_4x4_Blocks_Wide/High`, `Block_Width/Height`, `Size_Group`, `Intra_Mode_Context`, `Subsampled_Size` — from the additional-tables annex. A 5-agent adversarial review (syntax/read-order, CDF-selection contexts, per-value block-table diff, encode/decode inversion, hostile-input safety) confirmed it against the spec markdown with no findings. Feature-gated reads (segmentation/cdef/delta-q-lf/intrabc/palette) are deferred to later bites.
- **AV1 intra transform-size read** — spec section 5.11.15 (`read_tx_size`) + the `read_block_tx_size` intra branch (5.11.17) are authoritative (`src/av1_txsize.cyr`), with the `tx_depth` CDF-selection context from the parsing process (spec 9.3). `av1_read_tx_size` returns the block's luma `TxSize` (`Lossless` → `TX_4X4`; else start at `Max_Tx_Size_Rect[MiSize]` and apply `Split_Tx_Size` once per `tx_depth` when `MiSize>BLOCK_4X4 && allowSelect && TxMode==TX_MODE_SELECT`); the ctx is `(aboveW>=maxTxWidth)+(leftH>=maxTxHeight)` where, for an intra keyframe (`IsInters=0`), `aboveW`/`leftH` reduce to `Tx_Width/Height[InterTxSizes[neighbour]]`; the `maxTxDepth → Tx8/16/32/64` cdf-family dispatch consumes the 0.7.19 CDFs. `av1_write_tx_size` derives `tx_depth` back from the target `TxSize` (exact inverse). Tables `Max_Tx_Size_Rect` / `Max_Tx_Depth` / `Split_Tx_Size` from the additional-tables annex (cross-checkable against **libaom** `av1/common/common_data.h` `max_txsize_rect_lookup` / `sub_tx_size_map` and `av1/common/blockd.h`). A 4-agent adversarial review (read_tx_size fidelity, tx_depth ctx + cdf dispatch, per-value table diff, hostile-input safety) confirmed it against the spec with no findings. The inter `read_var_tx_size` tree and the `InterTxSizes` grid write are deferred to later bites.
- **AV1 transform-type derivation** — spec sections 5.11.48 (`transform_type` / `get_tx_set`) + 5.11.40 (`compute_tx_type`) are authoritative (`src/av1_txtype.cyr`), with the `intra_tx_type` CDF-selection context from the parsing process (spec 9.3). `get_tx_set` maps `(Tx_Size_Sqr, Tx_Size_Sqr_Up, is_inter, reduced_tx_set)` to the tx set (DCTONLY / INTRA_1 / INTRA_2, and the INTER_1/2/3 for faithfulness). `av1_transform_type_decode` reads `intra_tx_type` for a luma block when `set>0 && qindex>0`, via `av1_ncdf_intra_tx_set1/2[Tx_Size_Sqr[txSz]][intraDir]` (`intraDir` = `Filter_Intra_Mode_To_Intra_Dir[filter_intra_mode]` when filter-intra is used, else `YMode`), mapping through `Tx_Type_Intra_Inv_Set1/2`; `av1_transform_type_encode` is the exact inverse. `compute_tx_type` forces `DCT_DCT` for lossless/`txSzSqrUp>TX_32X32`, returns the luma value for plane 0, and for chroma uses `Mode_To_Txfm[UVMode]` gated by `Tx_Type_In_Set_Intra`. Spliced into the `coeffs()` loop (`src/av1_coeffs.cyr`) between `all_zero` and the eob position, retiring `PlaneTxType` as a caller input. Tables from the additional-tables annex + spec syntax; cross-checkable against **libaom** `av1/common/*` (`mode2txfm_map`, `av1_ext_tx_inv`, `av1_ext_tx_used`, `get_ext_tx_set`, `fimode_to_intradir`). A 5-agent adversarial review (transform_type fidelity, compute_tx_type/get_tx_set, per-value tables, the coeffs splice + Tx_Size_Sqr move, hostile-input safety) confirmed it against the spec with no findings. Inter chroma co-location + `Tx_Type_In_Set_Inter` and segmentation qindex are deferred to later bites.
- **AV1 residual driver** — spec sections 5.11.34 (`residual()`) + 5.11.36 (`transform_block` / `get_tx_size`) are authoritative (`src/av1_residual.cyr`), with the `clear_block_decoded_flags` process (5.11.3) for the `BlockDecoded` grid. `av1_transform_block` composes the already-tested intra primitives: `av1_intra_predict` (7.11.2) for the prediction (mode = `YMode` for luma, `UVMode` or `DC_PRED`+`av1_predict_chroma_from_luma` for a CfL chroma block), then — unless `skip` — `av1_coeffs_decode` (5.11.39, which computes the `PlaneTxType` internally) and, when `eob>0`, `av1_reconstruct` (7.12.3). It threads the `BlockDecoded` availability grid (the `haveAboveRight`/`haveBelowLeft` neighbour reads) and the luma `MaxLumaW/H` extents that CfL consumes. `av1_residual` iterates `transform_block` over each plane's tx grid (one 64x64 superblock chunk for the baseline). `av1_get_tx_size` picks the per-plane tx size (the chroma 64→32 clamp). Cross-checkable against **libaom** `av1/decoder/decodeframe.c` (`decode_reconstruct_tx` / `predict_and_reconstruct_intra_block` / `read_coeffs_txb`). A 5-agent adversarial review (transform_block orchestration, residual loop + get_tx_size, BlockDecoded + availability, coordinate/subsampling math, hostile-input safety) confirmed it against the spec with no findings. The 128x128-superblock chunk split, palette, inter `transform_tree`, and per-plane quantizer deltas are deferred.
- **AV1 partition tree** — spec sections 5.11.4 (`decode_partition`) + 5.11.5 (`decode_block`) are authoritative (`src/av1_partition.cyr`), with the partition / `split_or_horz` / `split_or_vert` CDF-selection from the parsing process (spec 9.3). `av1_decode_partition` reads the partition symbol (`av1_ncdf_partition_w8/16/32/64` by `bsl = Mi_Width_Log2[bSize]`; `split_or_horz`/`split_or_vert` use a binary CDF *synthesized* from the partition CDF by summing the vertical-/horizontal-splitting partition probabilities) and recurses / dispatches all 10 partition types via `Partition_Subsize`. `av1_decode_block` derives `HasChroma` + the availability flags, orchestrates `intra_frame_mode_info` → `read_block_tx_size` → `residual()`, and writes the `MiSizes`/`YModes`/`Skips`/`InterTxSizes` grids (clamped to the frame MI extent). The paired `av1_encode_partition`/`av1_encode_block` mirror the traversal so the tree round-trips. Cross-checkable against **libaom** `av1/decoder/decodeframe.c` (`decode_partition` / `decode_mbmi_block` / `read_tx_size`) and **dav1d** `src/decode.c` (`decode_sb` / `decode_b`). A 5-agent adversarial review confirmed the dispatch/ctx/symmetry with **one real finding — an out-of-bounds MI-grid write for an edge coding block on a non-superblock-aligned frame — fixed by clamping the write to the tile extent (spec-equivalent, since the reference decoders pad the MI arrays and the overhang cells are never read as neighbours)**. Partition W128 / 128x128 superblocks, palette, and segmentation are deferred. (`get_filter_type` is derived in `av1_residual` as of 0.7.26.)
- **AV1 tile / frame loop** — spec section 5.11.2 (`decode_tile`) + the `tile_group_obu` wiring (5.11.1) are authoritative (`src/av1_tile.cyr`), with `clear_above_context` / `clear_left_context` (7.x semantics) and the `init_coeff_cdfs` quantizer bucket (`base_q_idx` thresholds 20/60/120). `av1_decode_tile` runs `clear_above_context`, then per superblock row `clear_left_context` and per 64x64 superblock `clear_block_decoded_flags` + `decode_partition`. `av1_decode_intra_tile` builds the per-tile adaptive CDF contexts (`av1_ncdf_new` / `av1_ccdf_new(qbucket)`) + the symbol decoder (`init_symbol` / `exit_symbol`) and drives the loop into a `DrFrame`. The paired `av1_encode_tile` / `av1_encode_intra_tile` mirror it so a keyframe tile round-trips. Cross-checkable against **libaom** `av1/decoder/decodeframe.c` (`decode_tile` / `av1_init_above_context` / `av1_zero_left_context`) and **dav1d** `src/decode.c` (`dav1d_decode_tile_sbrow`). A 4-agent adversarial review (SB loop, CDF/symbol/qbucket wiring, clear-context, encode-symmetry + safety) confirmed it against the spec with no findings. **This closes the intra still-picture decode milestone — profile-0 AV1 keyframes decode end-to-end to pixels.** CDEF / loop-restoration / delta-q per-SB reads, multi-tile, and the frame-end CDF save/average + `decode_frame_wrapup` are deferred to the inter/conformance work.
- **AV1 deblocking loop filter** — spec sections 7.14.3 (filter size) / 7.14.4-5 (adaptive filter strength → lvl, limit, blimit, thresh) / 7.14.6 (sample filtering — mask, narrow 4-tap, wide low-pass) are authoritative (`src/av1_deblock.cyr`). `av1_lf_strength` derives the per-edge filter level from `loop_filter_level` + the ref/mode deltas (intra keyframe: `ref = INTRA_FRAME`, `mode_type = 0`); `av1_lf_limits` derives the thresholds from the level + `loop_filter_sharpness`; `av1_lf_mask` computes the hev/filter/flat/flat2 masks; `av1_lf_narrow` / `av1_lf_wide` are the 4-tap and 8/14-tap kernels; `av1_lf_sample` dispatches. The narrow filter's `filter1`/`filter2` use the sign-preserving `>>>` (spec's arithmetic `>>3` over a signed, frequently-negative intermediate). Cross-checkable against **libaom** `av1/common/av1_loopfilter.c` (`aom_lpf_*`, `filter4` / `filter8` / `filter14`) and **dav1d** `src/loopfilter_tmpl.c`. The edge loop + main driver (spec 7.14.1/2, added 0.7.28) — `av1_lf_edge` (dx/dy, onScreen, the `row|subY`/`col|subX` adjustment, `isBlockEdge`/`isTxEdge`/`applyFilter`, filter size + strength, the per-`MI_SIZE` sample loop) and `av1_deblock` (all vertical then horizontal boundaries per plane, in place) — walk the reconstructed frame's 4x4 edges, reading the `LoopfilterTxSizes` grid (written per tx block in `transform_block`) for the tx-edge / filter-size decisions. Two adversarial reviews (kernels; edge-loop + LoopfilterTxSizes/safety) verified it against the spec — the first independently flagged the narrow-filter arithmetic-shift (confirmed fixed).
- **rav1e** (Xiph) — encode-lane lineage; the 0.7.x encoder bring-up derives its structure from rav1e's keyframe path rather than libaom's, and the symbol-encoder byte/carry mechanics mirror `WriterBase` in `src/ec.rs`.

## H.265/HEVC

- **ITU-T Rec. H.265 (HEVC)** — authoritative. Sections used at 0.7.0: 7.3.1.2 / 7.4.2.2 (two-byte NAL unit header semantics), Table 7-1 (nal_unit_type), Annex B B.2.2 (byte-stream NAL unit decoding), 7.3.1.1 / 7.4.2 (emulation_prevention_three_byte), 7.3.3 / 7.4.4 (profile_tier_level incl. the 43+1 reserved constraint tail), 7.3.2.1 / 7.4.3.1 (VPS), 7.3.2.2 / 7.4.3.2.1 (SPS + conformance-window crop math), 7.3.2.3 / 7.4.3.3.1 (PPS), Table 6-1 (SubWidthC / SubHeightC), A.4.1 / A.4.2 + Table A.8 (level limits — the dimension bomb guard).
- **libde265** (`nal.cc`, `nal-parser.cc`, `vps.cc`, `sps.cc`, `pps.cc`) — cross-check for NAL header layout, EPB behavior (incl. trailing-EPB strip), and parameter-set field ordering. The library this family replaces.
- **ffmpeg** (`libavcodec/h2645_parse.c`, `hevc_ps.c`, `hevc_parse.c`) — second cross-check for Annex-B scanning and PS parsing; drishti is deliberately stricter (rejects non-zero bytes before start codes instead of resyncing).
- **HM reference model** — planned conformance-vector source for the 0.9.x battery (not yet consumed).
- Real-stream sanity anchor: parameter-set headers pack to the canonical on-wire bytes `40 01` (VPS) / `42 01` (SPS) / `44 01` (PPS), asserted in the suite.

## VP8/VP9

- **RFC 6386 — "VP8 Data Format and Decoding Guide"** — authoritative
  for VP8, and code-as-spec (it carries the reference implementation).
  Sections used at 0.7.0: 7.1 (range-subdivision theory), 7.3
  (bool_decoder / bool_encoder / read_literal / flush_bool_encoder /
  add_one_to_output — ported verbatim, then bounds-hardened), 9.1
  (uncompressed data chunk: frame tag, start code, dimensions), 9.2
  (version number semantics, 0-3 defined), 9.x sign-magnitude coding
  of signed header fields (e.g. 9.6 delta_q).
- **"VP9 Bitstream & Decoding Process Specification"** v0.7
  (Grange / de Rivaz / Hunt, Google) — authoritative for VP9. Sections
  used: 6.2 uncompressed_header(), 6.2.1 frame_sync_code(), 6.2.2
  color_config(), 6.2.3 frame_size(), 6.2.4 render_size(), 7.2 frame
  type semantics, 7.2.2 color_space enumeration, 9.2 bool decoding
  (identical coder to VP8's — one implementation serves both).
- **libvpx** — `vp8/decoder/dboolhuff.h` (`vp8dx_decode_bool`),
  `vp8/encoder/boolhuff.c` (`vp8_encode_bool`),
  `vp9/decoder/vp9_decodeframe.c` (`read_uncompressed_header`,
  `read_bitdepth_colorspace_sampling`) — cross-check for the coder
  arithmetic, carry propagation, and the VP9 header rejection set
  ("4:4:4 color not supported in profile 0 or 2"). The library this
  family replaces.
- **ffmpeg** — `libavcodec/vp8.c` / `vp9.c` header parsers — second
  cross-check of field order and the profile bit split.
